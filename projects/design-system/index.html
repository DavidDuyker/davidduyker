Design System

Material UI

We used a "land-and-expand" strategy to implement the first nZero design system. I lead our work focused on nailing the basics and building off Material UI.

Challenge

nZero needed a cohesive design system to scale their carbon accounting platform. With multiple product teams building features rapidly, we needed a systematic approach to design consistency without slowing down development.

Approach

We adopted a "land-and-expand" strategy that allowed us to:

  • Start with core components based on Material UI
  • Gradually expand coverage as teams adopted the system
  • Maintain backward compatibility during transition
  • Focus on high-impact, frequently used components first

Key Components

The initial system included:

  • Typography scale and text styles
  • Color palette with semantic tokens
  • Button variants and states
  • Form inputs and validation states
  • Data visualization components
  • Layout and spacing utilities

Results

The design system reduced design-to-development handoff time by 40% and improved consistency across the platform. Teams were able to build new features faster while maintaining visual coherence.
